Architecture

How LingoScribe works

A full-stack AI pipeline that takes raw audio in Urdu or Punjabi and returns structured transcription, English translation, and language insights in seconds.

Try it now
The three layers
01

Your Browser

lingoscribe-frontend.vercel.app

A zero-dependency static frontend served from Vercel. No framework, no build step. The browser collects audio, either a file you drop or a live recording via the MediaRecorder API, and sends it as a multipart HTTP request directly to the backend. All three API calls (/transcribe, /analyze, /ask) are plain fetch() calls over HTTPS.

  • Upload any audio format: WAV, MP3, OGG, FLAC, M4A, WEBM, AAC, WMA
  • Record live from microphone with real-time timer
  • Transcript table with timestamps, original text, English translation
  • AI insights rendered after transcription completes
  • Natural language Q&A about the transcript
02

Backend API

FastAPI · Docker · Render

A Python FastAPI app packaged in a Docker container and deployed on Render. It acts as a secure orchestration layer. The Groq API key never leaves the server. ffmpeg converts non-standard audio formats before forwarding to Groq. CORS is configured to accept requests only from the Vercel frontend domain.

  • /transcribe: receives audio, calls Groq Whisper, batch-translates segments
  • /analyze: sends transcript to Groq LLM, returns structured JSON insights
  • /ask: answers natural language questions about the transcript
  • /health: used by Render for liveness checks
03

Groq Cloud

api.groq.com

Groq provides both the speech model and the language model, on the same infrastructure, at very low latency. Transcription uses Whisper large-v3, OpenAI's most accurate open-source speech model. All text tasks (translation, insights, Q&A) use openai/gpt-oss-120b via the chat completions API.

  • Whisper large-v3: transcription with segment timestamps and language detection
  • GPT-OSS-120B: batch segment translation to English in one call
  • GPT-OSS-120B: summary, key topics, tone, and speech observation
  • GPT-OSS-120B: Q&A grounded in the transcript
Step-by-step workflow

From audio to insight in 9 steps

  1. 1
    Upload or record

    User drops an audio file or clicks the mic button to record. Any format is accepted and the browser passes it as-is.

  2. 2
    Send to backend

    The browser sends a POST /transcribe request with the audio as multipart form data over HTTPS.

  3. 3
    Format conversion

    If the format is not natively supported by Groq (e.g. WMA), ffmpeg converts it to MP3 before forwarding.

  4. 4
    Transcription on Groq

    The audio bytes are posted to Groq's Whisper large-v3 endpoint. The model detects the language (Urdu, Punjabi, etc.) and returns time-stamped segments.

  5. 5
    Batch translation

    All segment texts are sent to Groq's LLM in a single prompt. The model returns one English translation per segment as a JSON array, one API call for all segments.

  6. 6
    Response to browser

    Backend returns structured JSON: detected language, language probability, and the list of segments each with start/end times, original text, and English translation.

  7. 7
    Render transcript table

    The browser renders the segments as a table. Original Urdu/Punjabi text uses RTL-aware rendering; English translation shows in the adjacent column.

  8. 8
    AI insights

    The frontend automatically sends a POST /analyze with the full translated transcript. Groq LLM returns a JSON object with summary, key topics, tone, and a linguistic observation.

  9. 9
    Ask anything

    The user types any question about the transcript. A POST /ask sends the transcript and question together; the LLM answers from the transcript content only.

Security & privacy
HTTPS everywhere

All traffic between browser, backend, and Groq is encrypted in transit.

CORS restricted

The backend only accepts requests from the Vercel frontend domain. No third-party access.

API key server-side only

The Groq API key is stored as an environment variable on Render. Never sent to the browser or committed to GitHub.

No data stored

Audio files are written to a temporary directory and deleted immediately after processing. Nothing is persisted.
